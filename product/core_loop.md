# Core Loop

## Primärer Loop
```
Rausgehen → Vogel hören/sehen → App: bestimmen (Ton/Foto/manuell)
→ CACHEN → Catch gespeichert (Standort + Methode + Zeitstempel)
→ Belohnung: Animation + XP/Badge + Community-Karte
→ Freund-Benachrichtigung → Freund öffnet App → (Loop startet neu)
```

## Meta-Loop (Langfristbindung)
- **Bird-Dex füllt sich** — Silhouette → aufgedeckte Artenkarte, jeder Catch = Fortschritt
- **Tiefe nach dem Catch** — Artenkarte lesen, Gesang nochmal hören → Lernen passiert beiläufig
- **Progression** — häufige Arten → Lebensraum-Spezialisten → echte Raritäten
- **Streak** — mindestens einmal pro Woche cachen
- **Karte füllt sich** — die eigene Stadt wird über Zeit reicher
- **Jahreszeiten** — Zugvögel erzeugen FOMO („Mauersegler sind nur 4 Monate da")
- **Freunde** — wenn ein Freund eine neue Art cachet, motiviert es dich

## Sozialer Layer
- Catches standardmäßig öffentlich (Freunde / privat optional)
- Freunde sehen gegenseitige Catches im Feed + Karte
- Erste Person die eine Art in einem Gebiet loggt → Badge
- Keine kompetitiven Rankings — kollaborative Meilensteine („Hamburg hat 80 Arten geloggt!")

## Wiederkehr-Loop
- Push: „Ein Eisvogel wurde vor 800m vor 2h gesehen"
- Wöchentlich: „Du hast diese Woche 3 neue Arten gecacht"
- Saisonal: „Mauersegler sind in Hamburg angekommen"
- Freund: „Ben hat einen seltenen Buntspecht im Stadtpark gecacht"

## Design-Prinzipien
1. **Der Catch muss sich gut anfühlen.** Animation + Sound + Haptik = der wichtigste UX-Moment.
2. **Die App muss schön sein.** Hochwertige Illustrationen, warme Farben, sorgfältige Animationen.
3. **Tiefe ist optional, aber immer da.** Oberfläche (Name, Bild, ein Satz) + Tiefe (Verhalten, Nahrung, Zugwege). Wer taucht, findet mehr.
